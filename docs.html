Carousel

When creating a carousel please give it a unique id.

The HTML Structure should :

<div class="carousel" id="animals">
    <a href="#" class="pag prev" style="visibility:hidden">Previous</a>
    <div class="holder">
        <div class="viewer">
            <ul>
                <li><!-- Carousel item --></li>
                <li><!-- Carousel item --></li>
            </ul>
        </div>
        *<div class="jump"></div>       
    </div>
    <a href="#" class="pag next">Next</a>
</div>

The order of the <div> tags is not necessary. The necessary <div> tags are <div class="viewer">, <a href="#" class="pag prev"> and <a href="#" class="pag next">
*<div class="jump"> is optional if you want jump links (links at the bottom that go right to a slide in the carousel.

Activating the script

If you view the source of this page you will notice a script tag at the bottom of the page that is needed for the carousel. All you need to do is call ML.Carousel.setup({carousel:'unique-id', jump:true, increment:2}); The only really important one to fill out it is carousel. Jump is if you want jump links, by default it is set to false. Increment deals with the animation.

Tooltips & Modals

Be sure to follow the proper naming conventions when giving your element a unique rel attribute and id of the tooltip.
Example: <a rel="*&id"> = this may be ok, but you CAN NOT have an id with those characters.

They must meet this criteria:
-Begin with a letter A-Z or a-z
-Followed by: letters (A-Za-z), digits (0-9), hyphens ("-"), and underscores ("_").

Tooltips

There are two types of tooltips: static and dynamic. Static being the HTML is already on the page and Dynamic meaning it is created and put into the DOM.

You must add class "tooltip::" to the element you would like a tooltip for!
You do not have to use a link to activate the tooltip, it can be any element!

You can either use the default settings (defined in the JS) or you can set your own. To set your own custom settings you add them to the class of the element you want a tooltip for (i.e. <a href="#" rel="cows" class="tooltip::direction:top:width:100">). There are ONLY two settings you can define: direction (top, bottom, left, right) and width. Each setting is separated by a ":"

For Static Tooltips: You must place a unique rel attribute in the element you would like a tooltip for. This will be used to reference the tooltip id. See the example below for the HTML structure of the tooltip and what I mean in terms of referencing the tooltip.

<a href="#" class="tooltip::direction:top:width:140" rel="cows">I have a tooltip</a>

<!-- HTML STRUCTURE for Tooltip -->
<div id="cows" class="tooltip hidden">
    <span class="arrow"></span>
    <div class="content"><!--Place content here that you would like to be displayed--></div>
</div>

So you see rel and the id of the tooltip must be the same!

So, this example is saying that when I hover over the link a tooltip with an id of cows, will appear on top of the link and will be 140px in width.

For Dynamic Tooltips: All you need to do is place what you want displayed in the tooltip in the title attribute. So no need to add HTML structure for the tooltip or put a unique rel attribute in the element. You can really place anything in the title attribute, but if JS is disabled you will see your Operating System's default tooltip for elements with a title attribute. And if you have text that is bold, it will look like this: I am <strong>content</strong> inside a tooltip. With Javascript enabled it will look like: I am content inside a tooltip. The OS default tooltip can not render HTML code.

Modals

There are 3 types of modals: static, dynamic-ajax and dynamic-iframe. Static being the HTML is already on the page. Dynamic-ajax is a page being displayed via Ajax and Dynamic-iframe is for displaying only an iframe.

You must add class "modal::" to the link you would like a modal for!

You can either use the default settings (defined in the JS) or you can set your own .To set your own custom settings you add them to the class of the element you want a modal for (i.e. <a href="#" rel="cows" class="modal::width:100:height:100">). There are ONLY two settings you can define: height and width. Each setting is separated by a ":". There is another setting that is used for iframe modals (will mention below).

For Static Modals: You must place a unique rel attribute in the linkt you would like a modal for. This will be used to reference the modal id. See the example below for the HTML structure of the modal and what I mean in terms of referencing thetmodal.

<a href="" class="modal::width:140:height:200" rel="cows">Im a modal link</a>

<!-- HTML STRUCTURE for Modal -->
<div id="cows" class="modal hidden">
    <div class="header">
        <!-- Place your header text here, I currently am using an <h1> tag. -->
    </div>
    <div class="content">
        <!-- Place content in here -->
    </div>
</div>

So you see rel and the id of the modal must be the same!

So, this example is saying that when I click the link, the modal displayed has an id of cows, will appear centered on the page and will have a width of 140px and a height of 200px.

For Dynamic-iframe Modals: The page you want displayed in the iframe, place in the href attribute of the link. Give it a rel attribute of "iframe" and the header for the modal add to the class name (this will be added to the <h1> tag on the modal header. See below for proper link syntax:

<a href="http://mikewgd.com" rel="iframe" class="modal::width:200:height:100:header:Mikewgd.com">

For Dynamic-ajax Modals: The location of the file you would like to be Ajax'ed in goes in the href attribute of the link. Give it a rel attribute of "ajax" and the content for that page will be ajaxed in and will show up inside a modal. See below for proper link syntax:

<a href="ajax-files/hello.html" rel="ajax" class="modal::width:200:height:100">

Activating the scripts

If you view the source of this page you will notice a script tag at the bottom of the page that are needed for the tooltips and/or modals to work. To call the modals: ML.Modal.init(); and to call the tooltips: ML.Tooltip.init();

You can place these in either a window.onload or with the addLoadEvent which you will see when viewing the source of the page.

However since all settings are placed within a Javascript object you can esentially place them on a certain ele(link,span) by giving it a specific id or by putting an onclick on the ele(link,span) [I'de stay away from inline JS though].

This means you would not need to call the ML.Modal.init(); and/or ML.Tooltip.init(); Being able to do this allows you to change the events the functions are activated as well. So instead of click on a link to activate a modal you can make it happen when you hover over a link. Here is an example for both:

<a href="#" id="modal-example-link">link</a>
<a href="#" id="tooltip-example-link">link</a>

<script type="text/javascript">
    addLoadEvent(function() {
        _$('modal-example-link').onclick = function() {
            ML.Modal.show({id: 'modal-example', width: 400, height: 100});
        }

        _$('modal-example-link').onmouseover = function() {
            ML.Tooltip.show({id: 'tooltip-example', width: 400, direction: 'left'});
        }
     })
</script>
</body>

Toggler

There are two types of toggles: regular and slide. Regular is a basic hide and show of your content. Slide toggle slides the content up and down.

For Both toggles: You MUST give a unique rel attribute in your <a> tag. This will be to reference the element you would like to toggle. So follow the same rules when giving ids to elements. You MUST also add "toggle" to the class name of your <a> in order for either toggles to work.

I have also created an attribute for the link called "text" which you place on the <a>. This is the text you would like displayed when your content is displayed. See the example below:

<a href="#" class="toggle" rel="unique-id" text="Close It">Open It</a>

<div id="unique-id">
    <p>I am content to be toggled, yay!</p>
</div>

So the rel of the link must = the id you want to toggle!

So this example says its a regular toggle and we are toggling an element with an id of "unique-id". The text to open the toggle is "Open It" and when the content is displayed the link's text changes to "Close It".

For Slide Toggle: All you need to do is add "slide" to the link class name:

<a href="#" class="slide toggle" rel="unique-id" text="Close It">

Activating the script

If you view the source of this page you will notice a script tag at the bottom of the page that is needed for the toggler. All you need to do is call ML.Toggler.init();.

Slide Panel

Slide Panel is a lot like the slide toggle.

When you click on a tab, the content or "panel" slides down.

Follow the HTML structure below and you MUST give the panel a unique id!

<div class="curtain" id="unique-id">
    <div class="contents">
        <!-- Place content in here. Padding should be appended to the "contents" class -->
    </div>
    <a href="#" class="tab" text="up">down</a>
</div>

The "tab" or link to slide the panel up or down MUST be placed after the <div class="contents">

Place all margins and padding on <div class="contents"> as well.

If you choose to have the text change when the tab is fully opened and fully closed. Place the text you would like displayed when the panel is fully opened in the text attribute of the link. The text that you want displayed when the panel is fully closed place in the link.
EX: <a href="#" class="tab" text="shown when opened">shown when closed</a>

Activating the script

If you view the source of this page you will notice a script tag at the bottom of the page that is needed for the panels. All you need to do is call ML.SlidePanel.setup({id: 'unique-id1, unique-id2'});. Place the id or ids within single or double quotes. If more than one separate each by a comma and a space after the comma.

Accordion

There are two types of accordions: multi-toggler (toggle more than one at a time) and single-toggler (toggle only one at a time).

The HTML structure of the accordion (both types) should be like this and you MUST give a unique id:

<ul class="accordion" id="unique-id">
    <li>
        <a class="acc" href="#">link</a>
        <div class="content">
            <!-- Place content in here -->
        </div>
    </li>
</ul>

You may also choose to have one open when you arrive to the page. By placing "open" as the class name of the li element you want opened. Just a note: Only one should be opened for the single-toggler.

Activating the script

If you view the source of this page you will notice a script tag at the bottom of the page that is needed for the accordion. All you need to do is call ML.Accordion.init({id: 'unique-id', toggle: 1 or 0}). If you put 1 for toggle the elements will only toggle one at a time and 0 is a multi-toggle. If nothing is entered, the default is 0. If you want to make 2 accordions. You would do 'id, id, id'.

Fading Animation

This script will fade in and out elements. You MUST give the element a unique id! The script would be activated on any type of JS Event Handler. Such as a 'click', 'load', 'mouse over', 'mouse leave', and etc...

In order to use the script you must include the fading.js file.

When using the script there are several paramaters that you need pass through the object: 'ele', 'to', 'start', 'repeat' and 'cycle'. The ele is required, the last three (to, start, repeat and cycle) defaults will be used (defined in the JS).

If you have opacity set on an element be sure to place that number in 'to' and multiple by hundred. So if your element has opacity:.5, you would be 50 in 'to'. Consider the example below:

1 <a href="#" id="fade-link">Click me</a>
2
3 <div id="fade-test" style="opacity:.5;filter:alpha(opacity=50);">Faded element</a>
4
5 <script>
6   _$('fade-link').onclick = function() {
7      ML.Fading.prop({ele: 'fade-test', to: 0, start: 50, repeat: true, cycle: true});
8   }
9 </script>

1 - A link with an id of "fade-link" so we can reference to it later.

3 - This is the element we are fading and has an id of "fade-test". The filter:alpha(opacity=50); is for IE. All you do is take the opacity value in this case (.5) and multiple by 100 and you get 50.

6-8 - When you click on the link with an id of "fade-link" it will fade the element "fade-test" from .5(50) opacity to 0 and once it reaches opacity 0, it will than fade back to .5(50) opacity.

Repeat takes 2 options: true or false. When putting true when you hover/click/etc.. on the element it will go to the desired opacity and when you hover/click/etc.. on the element again it will go from the element's current opacity to the opacity the element's original opacity. When putting false when you hover/click/etc.. on the element it will go to the desired opacity and when you hover/click/etc.. on the element again it will stay at the current opacity and will not repeat the animation.

Cycle takes 2 options as well: true or false.When putting true when you hover/click/etc.. on the elment it will go to the desired opacity and then it will go back to the original opacity and when you hover/click/etc.. on the element it will do the same thing over again.

If you put repeat: false and put cycle: true, it will make cycle=false. Cycle relies on Repeat!

Tabs

Here is the HTML structure needed for creating tabs:

<ul class="tabs::holder:tab-contents">
    <li><a href="#unique">Tab Name 1</a></li>
    <li><a href="#unique">Tab Name 2</a></li>
</ul>
    
<div id="tab-contents">
    <div><!-- Tab Content 1 goes here [Tab Name 1 Contents] --></div>
    <div><!-- Tab Content 2 goes here [Tab Name 2 Contents] --></div>
</div>

The first thing is to create a <ul> element and be sure to add tabs:: to the class name. You will notice that the class name is: "tabs::holder:tab-contents"
- "tabs::" signifies that we are using tabs
- "holder:tab-contents" this is the id of the element that is holding each of the contents for our tabs
(going by the html above, the holder is the element with the id of "tab-contents"

Also be sure to give each tab link a unique href attribute beginning with a hash (#).

Here is a detailed example:

00 <ul class="tabs tabs::holder:tab-contents">
01     <li><a href="#fruits">Fruits I Like</a></li>
02     <li><a href="#veggies">Veggies I like</a></li>
03 </ul>
04
05 <div id="tab-contents">
06     <div class="content">
07         <h1>Fruits that I like</h1>
08         <p>Apples, Oranges, Pears, Cherries, Cranberries and Plums.</p>
09     </div>
10     <div class="content">
11         <h1>Veggies that I like</h1>
12         <p>Carrots, Tomatoes, Celery, Cucumbers and Radishes.</p>
13     </div>
14 </div>

00 - 03 is code need for the tabs. You MUST give the <ul> element a class of tabs:: and after that you put holder:IDofDivHolding tab contents i.e. #tab-contents.
By default 01 will be active and that means you will see 06-09 and 10-13 will be hidden.

So clicking on the link with the href=#"veggies" (line 02) will show lines 10-13 and 06-09 will be hidden.

Also, you will notice when you click on a tab and refresh the page it stays on that tab. This is why giving the links unique href attributes following a hash (#)
is very important!

Linking to Tabs:
- When linking to a tab from another page (i.e. a page that does not have the tabs on it): all you need to do is is append the unique hash on the link.
For Example: <a href="pageWithTabsOnIt.html#veggies">click</a> -- when clicked it will show the tab with the href of #veggies and will show/hide the appropriate
tab contents.
- Linking to a tab on the same page (i.e. a link on the page that has the tabs on them): once again you need to append the unique has on the link.
Correct Syntax: <a href="#veggies" target="internal">Click me</a>, I will show the tab with #veggies active and the appropriate content.

Activating the script

Pretty simple implementing this script. If you view the source of this page you will notice a script tag at the bottom of the page that is needed for this to work. All you need to do is call ML.Tabs.init(); And your good to go!
