Accordion

There are two types of accordions: multi-toggle (toggle more than one at a time) and single-toggle (toggle only one at a time).

HTML Structure

<ul class="accordion">
    <li>
        <a class="acc" href="javascript:void(0);">link</a>
        <div class="content">
            <!-- Place your content here -->
        </div>
    </li>
</ul>

Notes, Tips and Etc...

You may also choose to have an item open when you arrive to the page. Add "open" to the li class name. Just a note: Only one should be opened for the single-toggler.

When using a multi-toggle accordion add data-acc="multiple" to the ul element.

You can also access any item in an accordion by giving the ul a unique id. To link to the 2nd item in the accordion your url would be: http://yourwebsite.com/accordion.html#acc-{ID}-{1}


Activating the script

Include the accordion JavaScript file and all you need to do is call ML.AccordionHandler();. Doing this just loops through any accordion on the page and performs the necessary functions.

You can initialize it manually by doing:

var accordion = new ML.Accordion(ML.$('accEle'), false); 
accordion.init();

The second parameter must be include either true or false (multi-toggle). It can NOT be left blank.

Carousel

HTML Structure

The order of the HTML does not matter for this script. As long as they are children of the <div class="carousel" />. I would keep the slides in the unordered list structure.

<div class="carousel">
    <a href="javascript:void(0);" class="pag prev" rel="prv">&larr;</a>
    <a href="javascript:void(0);" class="pag next" rel="nxt">&rarr;</a>

    <div class="viewer">
        <ul>
            <li><!-- Slide Content --></li>
        </ul>
    </div>
</div>

Notes, Tips and Etc...

If you want jump links (i.e. the dots) just include <div class="jump"></div> as a child element.

There are 2 settings you can apply to the carousel: auto-rotate and current slide. All you need to do is add data-carousel="curr:2:rotate:true" to the <div class="carousel" /> (I.e Current slide will start on slide 3 and it will auto-rotate)

You may also choose to have a function(s) fire once the animation is complete and you can do this by applying a callback to the initialization of the carousel. Please look for details regarding this under the section Activating the script for the carousel.


Activating the script

Include the carousel JavaScript file and all you need to do is call ML.CarouselHandler();. Doing this just loops through any carousel on the page and performs the necessary functions.

You can initialize it manually by doing:

var carousel = new ML.Accordion(ML.$('carEle'), {curr: 1, rotate: true}, function(i, el){
    if (carousel.total == i) {
        alert('last slide');
        el.style.marginRight = '14px';   
    }
}); 

carousel.init();

Setting the current slide and auto-rotate are optional. Current slide defaults to 0 and rotate is set to false unless stated.

The callback function passes in the current slide and the carousel element which you can access in your callback function.

Overlays (Tooltips and Modals)

Tooltips & Modals

Be sure to follow the proper naming conventions when giving your element a unique rel attribute and id of the tooltip.
Example: <a rel="*&id"> = this may be ok, but you CAN NOT have an id with those characters.

They must meet this criteria:
-Begin with a letter A-Z or a-z
-Followed by: letters (A-Za-z), digits (0-9), hyphens ("-"), and underscores ("_").

Tooltips

There are two types of tooltips: static and dynamic. Static being the HTML is already on the page and Dynamic meaning it is created and put into the DOM.

You must add class "tooltip::" to the element you would like a tooltip for!
You do not have to use a link to activate the tooltip, it can be any element!

You can either use the default settings (defined in the JS) or you can set your own. To set your own custom settings you add them to the class of the element you want a tooltip for (i.e. <a href="#" rel="cows" class="tooltip::direction:top:width:100">). There are ONLY two settings you can define: direction (top, bottom, left, right) and width. Each setting is separated by a ":"

For Static Tooltips: You must place a unique rel attribute in the element you would like a tooltip for. This will be used to reference the tooltip id. See the example below for the HTML structure of the tooltip and what I mean in terms of referencing the tooltip.

So you see rel and the id of the tooltip must be the same!

So, this example is saying that when I hover over the link a tooltip with an id of cows, will appear on top of the link and will be 140px in width.

For Dynamic Tooltips: All you need to do is place what you want displayed in the tooltip in the title attribute. So no need to add HTML structure for the tooltip or put a unique rel attribute in the element. You can really place anything in the title attribute, but if JS is disabled you will see your Operating System's default tooltip for elements with a title attribute. And if you have text that is bold, it will look like this: I am <strong>content</strong> inside a tooltip. With Javascript enabled it will look like: I am content inside a tooltip. The OS default tooltip can not render HTML code.

Modals

There are 3 types of modals: static, dynamic-ajax and dynamic-iframe. Static being the HTML is already on the page. Dynamic-ajax is a page being displayed via Ajax and Dynamic-iframe is for displaying only an iframe.
